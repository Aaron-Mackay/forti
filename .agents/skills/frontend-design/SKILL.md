This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics.

## Context

Load [shared prep](reference/shared-prep.md) before any design work. It defines the required context, gathering order, and the rule against inferring tone from the codebase.

## Design Direction

Commit to a clear aesthetic direction, then implement working code that is production-grade, visually distinct, cohesive, and precise.

## Shared Standards

- Prefer strong hierarchy over decorative noise.
- Avoid generic AI tells: cyan-on-dark gradients, glassmorphism everywhere, hero metric templates, repeated card grids, and default system fonts.
- Use motion sparingly and purposefully.
- Make every interactive surface feel intentional and responsive.
- Adapt the interface to the context instead of only shrinking it.
- Keep copy short and purposeful.

## The AI Slop Test

If someone saw the result and immediately said "AI made this," the design is too generic. Aim for something that feels intentionally authored instead of templated.

## Implementation Principles

Match implementation complexity to the aesthetic vision. Keep the code as deliberate as the UI.
