# Shared Design Prep

Use this before any design-oriented skill in this repo.

## Required context

Collect, at minimum:

- Target audience
- Primary use cases
- Brand personality / tone

Some skills need extra context; keep those requirements in the skill-specific file.

## Context gathering order

1. Check the loaded instructions first. If a **Design Context** section already exists, proceed.
2. Check `.impeccable.md` at the project root. If it exists and contains the required context, proceed.
3. If neither source has context, run `teach-impeccable` before doing any design work.

## Rules

- Do not infer design context from the codebase.
- Do not skip context gathering because the implementation seems obvious.
- When a skill points to a focused reference, load only the reference that matches the task.
